对比

BST vs AVL · 为什么要平衡

二叉排序树 BST

动态、可能退化
  • 查找 O(h),h 受插入顺序影响
  • 有序输入会退化成链表,h = O(n)
  • 实现简单,适合随机数据
VS

平衡二叉树 AVL

自平衡、稳定高效
  • 任意结点左右子树高差 ≤ 1
  • 查找/插入/删除恒为 O(log n)
  • 代价:插入后需旋转调整

对比说明:BST 的退化风险 vs AVL 的稳定 O(log n)。平衡是关键词。

演进

查找结构的演进

阶段一

顺序 / 折半查找

静态表上的线性与二分思想,折半要求有序。

阶段二

二叉排序树

把查找搬到树上,支持动态插入与删除。

阶段三

AVL / 红黑树

引入自平衡,把最坏高度压到 O(log n)。

阶段四

B 树 / 散列

面向外存索引与平均 O(1) 的直接定址。

主线从"静态顺序"走向"动态平衡",核心是控制高度。
考点每种结构记定义 + 复杂度 + 适用场景三件套。

把查找这根线串起来:每个阶段解决了上一阶段的什么痛点。

算法流程

AVL 插入后的再平衡流程

1

插入

按 BST 规则插入新结点。

2

回溯更新

自底向上更新各结点高度。

3

判失衡

平衡因子 ∉ {−1,0,1} 即失衡。

4

旋转调整

LL/RR/LR/RL 四情形对应旋转。

易错四种失衡的判定与旋转方向是高频考点,务必画图分清。

逐步讲:插入 → 更新 → 判定 → 旋转。重点是四种失衡情形。
